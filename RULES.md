# Дополнительные правила работы с кодом

## 1. Безопасность и управление секретами
* [cite_start]Долгосрочные токены доступа (Long-Lived Access Tokens) и учетные данные GitHub никогда не должны зашиваться (hardcode) в сам скрипт.
* [cite_start]Все токены, пароли и IP-адреса выносятся в файл `.env` или отдельный `config.yaml`.

---

## 2. Режим "Холостого прогона" (Dry Run)
* [cite_start]При работе с реестрами Home Assistant скрипт должен поддерживать флаг `--dry-run` или тестовый режим.
* [cite_start]В этом режиме скрипт только выводит в консоль JSON-запросы, которые собирается отправить, но не выполняет реальную отправку.
* [cite_start]Это убережет от создания дублей в Home Assistant при отладке.

---

## 3. Структурированное логирование
* [cite_start]Вместо обычных вызовов `print()` используем модуль `logging`.
* [cite_start]Это поможет отлаживать скрипты при автоматическом запуске: `INFO` используется для успешного выполнения задач (например, создания комнаты), `ERROR` — для фиксации ошибок (например, если WebSocket вернул отказ или файл не прочитался)[cite: 25].

---

## 4. Фиксация зависимостей
* [cite_start]Учитывая работу со сторонними библиотеками вроде `pandas` и `websockets`, необходимо всегда обновлять список зависимостей (например, файл `requirements.txt`) при внедрении новых модулей[cite: 26].
